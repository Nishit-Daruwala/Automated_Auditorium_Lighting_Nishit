# 📁 Understanding Other Folders (Non-Phase Directories)

While the `phase_1` through `phase_8` folders contain the core logical steps of the Automated Auditorium Lighting pipeline, the project relies on several other crucial support directories to function smoothly, provide user interfaces, and store data. 

Here is a breakdown of every other folder in the project:

### `api/`, `static/`, and `templates/`
- **What they do:** These three folders make up the web components for the **Phase 5 (3D Visualizer)**. `api/` contains the routing logic and WebSockets, `templates/` holds the base HTML, and `static/` holds the CSS and JavaScript needed for the Three.js 3D rendering.
- **Why we need it:** So the root `app.py` visualizer can serve a functional webpage to user's browser.
- **Connections:** Connected to `app.py` and `phase_5/`.

### `backend/` and `frontend/`
- **What they do:** This is the **Main User Web Application**. `frontend/` is a modern React/Vite UI where a user can upload a script and click "Process." `backend/` is the dedicated FastAPI server that receives that script and triggers the `main.py` pipeline.
- **Why we need it:** It provides a beautiful, user-friendly graphical interface (GUI) so non-technical directors don't have to use the command line terminal to use the system.
- **Connections:** `frontend/` talks to `backend/`, which in turn executes `phase_1` through `phase_4`.

### `Phase-7-frontend-extracted/`
- **What it does:** A specialized, standalone React dashboard (The Evaluation Dashboard).
- **Why we need it:** To visually display the "8-Check System" (SCH, HRD, CFT, DRF, etc.) metrics calculated during Phase 7 in a highly readable, premium UI.
- **Connections:** Reads the evaluation summary JSON files generated by the `Evaluation_and_accuracy/` scripts.

### `Evaluation_and_accuracy/`
- **What it does:** The Python scripts that perform the heavy mathematical calculations for Phase 7. It checks the generated lighting cues against the story's baseline to find drift, logic clashes, or hallucinations.
- **Why we need it:** To give the AI's work a mathematically solid "Report Card" before letting it touch real auditorium lights.
- **Connections:** Evaluates the outputs generated by `phase_4/` and `phase_6/`.

### `contracts/`
- **What it does:** Holds JSON Schema definition files (like `scene_schema.json`).
- **Why we need it:** These are the strict "Rulebooks" for data formatting. If Phase 1 tries to output data that doesn't match the `contracts/` rulebook, the system catches the error immediately instead of crashing later.
- **Connections:** Heavily used by `validation_layer.py` in Phase 1 and the validators in Phase 6.

### `data/`
- **What it does:** The central storage hub for the project. It contains subfolders for uploaded raw scripts, intermediate processed JSONs, FAISS vector databases, and the final generated lighting cues.
- **Why we need it:** Without it, the project would have nowhere to save its work or read user inputs.
- **Connections:** Every single phase reads from or writes to the `data/` directory.

### `docs/`
- **What it does:** Contains raw textbook PDFs, technical manuals, and stage lighting literature.
- **Why we need it:** These files were scanned by text-extraction scripts to build the "Lighting Semantics" Knowledge Base (RAG) in Phase 3. It's the source of the AI's lighting knowledge.
- **Connections:** Connected specifically to `phase_3/extract_book_rules.py`.

### `models/`
- **What it does:** Stores local, downloaded Machine Learning model files (like DistilRoBERTa) and their state configurations.
- **Why we need it:** If the cloud AI (OpenAI/HuggingFace) goes offline, the system falls back to these models residing locally on the hard drive to ensure the show goes on.
- **Connections:** Primarily utilized by `phase_2/emotion_analyzer.py` and local text generators.

### `utils/`
- **What it does:** A folder full of small, reusable "helper" Python scripts (e.g., `file_io.py`, `llm_client.py`). 
- **Why we need it:** Instead of writing the code to "open a file" or "ping an API" 50 times across 8 phases, developers write it once in `utils/` and share it everywhere.
- **Connections:** Imported aggressively by almost every phase in the pipeline.

### `tests/`
- **What it does:** Contains automated testing scripts (Pytest files).
- **Why we need it:** Whenever developers add new code, they run the `tests/` folder to securely verify they haven't accidentally broken existing features.
- **Connections:** Tests logic from all phases, primarily focusing on core functionality.

### `EN_EX/`
- **What it does:** Contains internal English/Hindi documentation files breaking down exactly which Python functions belong to which phases.
- **Why we need it:** Serves as a technical manual for developers navigating the complex codebase.
- **Connections:** Documents `phase_1` through `phase_8`.
