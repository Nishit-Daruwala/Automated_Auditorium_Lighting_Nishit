# Phase 2: AI Emotion & Narrative Analysis

## Workflow
Phase 2 (also known as the Scene Analyzer) takes cleanly segmented scenes from Phase 1 and utilizes Large Language Models (LLMs)—specifically `gpt-4o-mini` via OpenAI's API—to infer emotional intelligence, audience tone, and narrative context from the raw script.

1. **Global Anchor Context (Phase 2A):** First, the original full script is sent into the LLM to write a concise 2-paragraph narrative "global scope summary" covering the overarching plot and main character arcs.
2. **Multi-Head Sequential Window (Phase 2B):** A "sliding window" context state is established to eliminate AI hallucination and retain character memory. The LLM handles one scene at a time. It gets:
   - The Global Narrative Context.
   - A short summary of the previous scene.
   - The current scene's raw text.
3. **Emotion & Narrative Extraction:** The LLM returns a strictly formatted JSON payload denoting:
   - Primary emotion (e.g., Joy, Tension, Fear) and confidence score.
   - Secondary and Accenting emotions.
   - Mood Shift mechanics (gradual vs. sudden).
   - "V3 Metrics" such as Irony index, scene energy score, and comedy intensity.
4. **Narrative Arc Pipeline (Phase 3 Hook):** It automatically identifies where this scene belongs in the classical plot structure (e.g., rising action, climax, comic relief).
5. **Graph RAG Generation:** As scenes are analyzed, spatial and temporal relationships are written to a Graph model (`build_scene_graph`) so global narrative dynamics are mapped.

## How it Connects
Phase 2 converts cold text strings into **Structured Emotional Payload Data**. It feeds directly into Phase 3 (RAG/Knowledge Layer) and Phase 4 (Lighting Decision). Phase 3 will search for historical lighting examples based on the output of Phase 2, and Phase 4 translates the emotions generated by Phase 2 into rigid DMX/Lighting parameter groups (Intensity, Colors, Angles).
